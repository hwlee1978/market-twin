2. Market Twin 엔진 4-Layer Architecture

데이터 → 페르소나 → 합의 → 출력. 각 레이어는 독립 모듈이며, Tier가 올라갈수록 LLM 다양성과 시뮬 개수가 증가.

📊 4-Layer Simulation Pipeline

Layer 1 · Reference Anchor (정부 OpenData)
국가/카테고리 ground truth — DART, KOTRA, Comtrade, 관세청, World Bank, Hofstede, MFDS, aT
packages/shared/src/market-research/*.ts · packages/shared/src/reference/hofstede-dimensions.ts
▼
Layer 2 · Persona Generation (LLM × N)
국가별 소득 분포 + 문화 차원 → 200명 Persona 생성. 채택의사·지불의향·반대이유 추론.
simulation/runner.ts · simulation/income-distribution.ts · llm/anthropic|openai|gemini|deepseek.ts
▼
Layer 3 · Ensemble Synthesis (N개 시뮬 합의)
Round-robin Provider 배정 → Best Country 분포 + Cross-LLM Agreement Score 산출.
simulation/orchestrator.ts · simulation/ensemble-narrative.ts · calibration/provider-weights.ts
▼
Layer 4 · Output & Delivery
대시보드 시각화, PDF/CSV, 공유 토큰, Persona-chat (개별 페르소나와 대화).
app/[locale]/(app)/projects/[id]/results · api/ensembles/[id]/pdf · api/persona-chat

4. Multi-LLM 전략 5 Provider × 4 Stage

단일 LLM 편향 제거가 핵심. Stage별 최적 모델 + Failover로 503 spike 대응.

단계 역할 Primary Failover 비고
Persona Generation 200명 가상 소비자 생성 Anthropic Sonnet 4.6 OpenAI GPT-4o, DeepSeek r1 Round-robin 분배 (Decision+)
Persona Reaction 채택/거부 의사 추론 Round-robin 3 Provider — 편향 분산 핵심 단계
Synthesis N개 결과 → 최종 추천 Gemini 2.0 Flash → Anthropic auto-failover 503 대응 shipped (f27d2d9 직후)
Structured Tasks JSON 추출·검증·요약 Anthropic Haiku 4.5 OpenAI GPT-4o-mini 비용·속도 최우선

🔄 Round-Robin

시뮬 1·4·7 = Anthropic, 2·5·8 = OpenAI, 3·6·9 = DeepSeek. 시뮬 간 Provider 편향 분산.

⚠️ 자동 Failover

Gemini synthesis 503 spike 자동 Anthropic 폴백. Honest snapshot attribution 보존.
