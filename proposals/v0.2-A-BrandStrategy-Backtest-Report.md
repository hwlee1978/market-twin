## 1. 왜 이 backtest를 했나

어제 K-뷰티 방법론 v3에서 발견한 3가지 defect 중 하나가 **Tirtir-class anchor blind spot** — KOL/D2C-native 브랜드가 카테고리 평균 anchor에 묻혀 wrong country로 추천되는 패턴. 가설은 "사용자가 brand strategy 힌트 (founder origin / channel priority / KOL ecosystem)를 직접 입력하면 LLM이 macro anchor보다 더 weighted하게 사용해서 Tirtir-class miss를 잡을 수 있다"였음.

v0.2-A는 그 가설의 첫 product 구현 — 위저드에 collapsible 섹션 + 3 nullable 컬럼 + ProjectInput 로드 wiring + 3개 LLM 프롬프트(country / marketProfile / synthesis)에 `brandStrategyBlock()` 주입.

이 리포트는 v0.2-A가 실제로 가설을 검증하는지 측정한 결과 + 정직한 한계.

## 4. 핵심 발견 3가지

### 4.1 brandStrategy 힌트는 prior를 움직임 (non-zero signal)

3 brand 중 2개(Anua, Tirtir)에서 recommendation이 다른 country로 이동. v0.2-A는 사용자 입력을 무시하지 않고 실제로 LLM 판단에 영향을 줌. 다만 *움직임의 방향*은 사용자 의도와 안 맞을 수 있음 (사용자가 "JP가 답일 것 같다"는 직관을 입력해도 LLM은 다른 방향으로 해석할 수 있음).

### 4.2 BoJ는 정답(US) 유지 — no-regression 확인

기존에 정답이었던 BoJ가 brandStrategy 힌트를 추가했을 때도 US를 유지. v0.2-A 출시가 기존 정답을 깨뜨리는 부작용은 없음. ship-safe.

### 4.3 Tirtir-class miss는 v0.2-A로 못 풂

가장 중요한 발견. v0.2-A의 원래 동기였던 Tirtir-class miss (KOL-native 브랜드의 JP-via-influencer 성공 패턴)가 1-sim/3-sim multi-LLM 양쪽에서 모두 JP를 안 잡음. 진단:

- 진짜 Tirtir 성공은 "JP TikTok ASMR 인플루언서 + 일본 매장 입점 결합"이라는 **market-specific KOL signal** 덕분
- 우리가 받는 brand-strategy 입력은 founder bg + channel priority + kol relationships로, 시장과 무관한 **macro** 차원
- 즉 "이 브랜드는 KOL이 강하다"는 정보로는 "**어느 나라**가 그 KOL 모델과 가장 잘 맞는가"를 LLM이 판단할 anchor가 부족
- anchor 차원의 본질적 한계 — input field 추가만으로는 해결 안 됨

## 5. 정직 reframe

### 5.1 v0.2-A는 무엇을 풀고 무엇을 못 푸는가

**풀음:**
- 사용자가 가진 brand-specific 컨텍스트를 시뮬에 주입할 channel을 만들었음 (없을 때 시뮬은 description text만 봄)
- BoJ-style 케이스(이미 정답)에서 추천을 *강화*함
- description 강도 가이드와 함께 제공해서 "약한 description → 약한 시뮬" 피드백 loop 형성

**못 풀음:**
- Tirtir-class miss (anchor blind spot for market-specific KOL ecosystem) — v0.2-A 입력 차원으로는 부족
- 잘못된 STRONG → 올바른 STRONG 변환 — 잘못된 STRONG → 다른 잘못된 STRONG (Anua, Tirtir)으로 이동

### 5.2 BD pitch 영향

K-뷰티 v3 BD pitch에 v0.2-A를 추가할 때:

- ✅ "사용자 brand-context 입력을 받아서 시뮬 prior에 반영" 정도의 incremental story
- ✅ "기존 정답 케이스 regression-free" no-harm claim
- ❌ **"Tirtir-class miss를 풀었다"는 주장은 하면 안 됨** — backtest가 직접 반증

대신 더 강력한 신뢰 자산: "v0.2-A를 출시했지만 Tirtir-class miss는 안 풀린다는 사실을 같은 날 자체 backtest로 확인했고 정직하게 공개합니다" — methodology v3와 동일한 정직 evidence 모티프.
