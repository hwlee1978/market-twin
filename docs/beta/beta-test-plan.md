# Market Twin 베타 테스트 설계서

> 작성일 2026-06-18 · 대상 제품: **Market Twin** (한국 제품의 해외 진출을 출시 전에 검증하는 AI 시장 시뮬레이션)

---

## 1. 목적 — 무엇을 검증하는가

베타의 목표는 "사용자를 모으는 것"이 아니라 **출시 전에 다음 4개 가설을 검증**하는 것이다.

| # | 가설 | 검증 방법 |
|---|---|---|
| H1. 가치 | 사용자는 "출시 전 해외시장 반응을 시뮬레이션으로 미리 본다"는 가치를 이해하고 원한다 | 데모 실행률, 실제 프로젝트 전환율, 만족도 설문 |
| H2. 결과 신뢰 | 시뮬레이션 결과(추천 국가·페르소나·가격)가 "실제로 의사결정에 쓸 만하다"고 느낀다 | 결과 만족도(5점), "이 결과로 무엇을 하겠는가" 인터뷰 |
| H3. 완주 | 비기술자도 가입→위저드→결과까지 막힘 없이 완주한다 | 단계별 이탈률(퍼널), 첫 시뮬 완주율 |
| H4. 운영 안정 | 가입·메일·시뮬레이션 파이프라인이 실사용에서 안정적으로 돈다 | 가입 성공률, 메일 전달률, 시뮬 실패율(타임아웃 포함) |

→ 베타 종료 시 위 4개에 대해 "간다 / 고친다 / 접는다"를 데이터로 판단한다.

---

## 2. 베타 단계 (2단계, 약 4주)

비공개로 시작해 신뢰가 쌓이면 공개로 확대한다. 한 번에 공개하지 않는 이유: 초기엔 **깊은 피드백**이 숫자보다 중요하고, 비용·전달률 등 운영 리스크를 작은 규모에서 먼저 잡기 위함.

### Phase 1 — 비공개 베타 (1~2주차)
- **규모**: 20~30명 (지인·타겟 고객·액셀러레이터 네트워크에서 직접 초대)
- **목표**: H2(결과 신뢰)·H3(완주) 중심. 1:1 인터뷰로 깊게 판다.
- **모집**: 초대 링크 + 개별 메일/메시지. `/beta` 랜딩으로 유도.
- **성공 기준(게이트)**: 첫 시뮬 완주율 ≥ 70%, 치명 버그 0, 결과 만족도 평균 ≥ 3.5/5 → 충족 시 Phase 2.

### Phase 2 — 공개 베타 (3~4주차)
- **규모**: 제한 없음(다만 비용 가드 — §9)
- **목표**: H1(가치)·H4(운영 안정) 중심. 트래픽·전환 퍼널 확보.
- **모집**: 링크드인, 스타트업/수출 커뮤니티, KOTRA·무역 네트워크, 콘텐츠(블로그/사례).
- **성공 기준**: 주간 활성 사용자 추세 우상향, 데모→실제 전환율 ≥ 20%, NPS ≥ 0.

## 3. 베타 제품 사양 (현재 셋업 — 이미 적용됨)

| 항목 | 사양 | 비고 |
|---|---|---|
| 가격 | **완전 무료** | 결제 경로는 노출되나 강제 없음 |
| 무료 한도 | **7일 또는 초기검증(Hypothesis) 2회** (먼저 도래 시점) | `BETA_TRIAL_SIMS=2` |
| 샘플 데모 | **하루 3회** (무료 한도와 별개) | 가입 유도용, Haiku로 비용 ~$0.05/회 |
| 가입 | 이메일·구글 + **이메일 확인 필수** | 확인 메일 재발송 지원 |
| 발신 메일 | `noreply@markettwin.ai` (Resend, 도메인 인증 완료) | 확인 메일 + 베타 환영 메일 |
| 시뮬 티어 | 무료 = **초기검증(Hypothesis)** = 600 페르소나 × 3-LLM | 결과: Top-2 추천 시장 + 페르소나 + 가격 + PDF |
| Mr.AI(마케팅 자동화) | **비공개** | `NEXT_PUBLIC_MRAI_ENABLED=false` |

## 4. 타겟 테스터 & 모집

### 누구
- 한국 제품을 **해외에 팔려고 준비 중**인: 중소기업·스타트업 대표/마케터, 수출 담당자, 무역상사, D2C 브랜드, 액셀러레이터·VC 포트폴리오사.
- 공통점: "어느 나라부터?", "가격은 얼마로?", "현지 반응은?"을 **출시 전에** 알고 싶은 사람.

### 어디서 (채널 우선순위)
1. **직접 초대** (Phase 1): 네트워크 내 적합 고객 20~30명.
2. **링크드인**: 수출·글로벌 진출 키워드 타겟 포스팅 + DM.
3. **커뮤니티**: 스타트업(디스콰이엇·EO), 무역·수출 카페, 업종 단톡.
4. **기관 네트워크**: KOTRA, 창업진흥원, 수출바우처 수행사.
5. **콘텐츠**: "해외진출 검증" 사례·방법론 글 → `/beta` 유입.

### 메시지(한 줄)
> "감으로 나라 고르지 마세요. 출시 전에 600명의 현지 가상 소비자에게 먼저 물어보세요. 베타 무료."

## 5. 성공지표 (KPI)

| 구분 | 지표 | 베타 목표 |
|---|---|---|
| 획득 | `/beta` 방문 → 가입 전환율 | ≥ 15% |
| 활성화 | 가입 → 첫 시뮬(데모 or 실제) 완주율 | ≥ 70% |
| 핵심행동 | 데모 → 실제 프로젝트 전환율 | ≥ 20% |
| 참여 | 1인당 평균 시뮬 횟수(데모 제외) | ≥ 1.3 |
| 품질 | 결과 만족도(완료 후 설문, 5점) | ≥ 3.8 |
| 추천 | NPS | ≥ 0 (공개 베타 기준) |
| 안정 | 시뮬 실패율(타임아웃 포함) | ≤ 5% |
| 안정 | 확인 메일 전달률(Resend Delivered) | ≥ 98% |

→ 측정: PostHog(퍼널/이벤트) + Resend 로그(메일) + Supabase(가입/시뮬) + 설문.

## 6. 피드백 수집 체계

여러 경로로 받되, **정성(인터뷰)**과 **정량(설문·이벤트)**을 분리한다.

1. **완료 후 마이크로 설문** (정량): 시뮬 결과 화면에서 "이 결과가 의사결정에 도움이 되나요?"(5점) + 자유 한 줄.
2. **이메일 피드백** (상시): 환영 메일·푸터에 `contact@markettwin.ai` 안내(이미 반영).
3. **1:1 인터뷰** (정성, Phase 1 핵심): 활성 사용자 5~8명, 30분. "결과로 무엇을 하겠는가"를 묻는다.
4. **세션 관찰** (선택): PostHog 세션 리플레이로 위저드 이탈 지점 확인.
5. **버그 채널**: 별도 폼 또는 이메일. 심각도(치명/주요/사소) 분류.

> 매주 1회 피드백을 모아 "가설별로 분류 → 다음 주 우선순위" 회고.

## 7. 일정 (주차별)

| 주차 | 활동 | 산출물/게이트 |
|---|---|---|
| **0주 (준비)** | 셋업 완료·QA·테스터 매뉴얼·`/beta` 랜딩 | ✅ 대부분 완료, 랜딩 제작 중 |
| **1주** | Phase 1 초대(20~30명), 온보딩, 1:1 인터뷰 시작 | 첫 퍼널 데이터, 인터뷰 3건 |
| **2주** | 피드백 반영(핫픽스), Phase 1 게이트 점검 | 게이트 충족 여부 결정 |
| **3주** | Phase 2 공개 모집 시작(링크드인·커뮤니티) | 트래픽·전환 퍼널 |
| **4주** | 데이터 종합, 정식 전환 의사결정 | 베타 리포트 + Go/No-Go |

## 9. 리스크 & 완화

| 리스크 | 영향 | 완화 |
|---|---|---|
| LLM 비용 폭증(공개 베타 남용) | 비용 | 무료 2회/7일 + 데모 하루 3회(Haiku) + trial-abuse 한도. 일일 비용 모니터링, 급증 시 캡 하향 |
| 확인 메일 미수신/스팸 | 가입 이탈 | Resend 도메인 인증(완료) + 재발송 버튼. Delivered율 모니터링 |
| 시뮬 결과 신뢰 부족 | 핵심 가치 실패 | 인터뷰로 "왜 못 믿는지" 파악 → 근거·출처 노출 강화 |
| 무료 한도 소진 후 이탈 | 리텐션 | 베타엔 한도 상향 여지(env로 즉시 조정 가능) |
| 시뮬 타임아웃 | 결과 못 봄 | 무료 티어 Haiku로 ~300s(800s 한도 내). worker 도입은 정식 전 검토 |

## 10. 종료 / 정식 전환 기준

베타 종료(4주차) 시 **Go**(정식 준비) 판단 기준:

- H2(결과 신뢰): 만족도 ≥ 3.8 **그리고** 인터뷰에서 "이 결과로 OO하겠다"는 구체 행동 3건 이상
- H3(완주): 첫 시뮬 완주율 ≥ 70%
- H4(안정): 시뮬 실패율 ≤ 5%, 메일 전달률 ≥ 98%
- 정성: "돈 내고도 쓰겠다"는 의향 표명 5건 이상

**Go 시 정식 전환 작업**(별도): 결제 갱신 cron 완성, 유료 플랜 가격 확정, billing "1건/Free Trial" → 정식 표기, worker 도입 검토, 무료 한도 재설계.
